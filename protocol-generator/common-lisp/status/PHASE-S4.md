## The grind: 7 → 0 FAIL (iteration count: 2 oracle runs)

First run: **568 / 277P / 195W / 7 FAIL / 89S**. The 7 FAILs were CL **code bugs**,
fixed by deriving the correct behavior from V7 + the cohort (never by doctoring the
oracle, hand-editing vectors, or disguising a FAIL as a SKIP):

1. **`hex` was uppercase, the address-space path convention is lowercase**
   (A-CL-009 ⚑) — broke the §3.4/§3.5 grant-signature invariant-pointer path
   (`core_register_grant_signature_at_invariant_path` + unregister symmetry) and the
   §5.1 revocation marker path (`revoke_happy_path_writes_marker` +
   `revoked_cap_denied_on_use`). Internally consistent (S3 CL-to-CL loopback passed),
   but case-mismatched against the oracle's lowercase tree-path keys. Fixed
   `peer-model.lisp` `hex` → lowercase. **Escalated to arch**: V7 §3.4/§3.5 SHOULD
   state hex-case explicitly — a latent interop trap for any peer whose stdlib hex
   defaults to uppercase.

2. **§PR-8 / V2(a) cross-peer dispatch boundary** — `check-resource-scope`
   canonicalized a cap's grant resource patterns against the LOCAL frame; a
   peer-local bare-`*` cap presented cross-peer was wrongly ACCEPTED
   (`captok_form_dispatch_minted_pl_presented_xpeer`). Fixed to canonicalize the
   grant's own patterns against the **granter's** peer_id (resolved at the dispatch
   site, `resolve-granter-peer-id`). The S3 code's "exercised at S4" comment was a
   stub — now actually implemented (cohort V2(a) shape).

3. **§5.7 delegation caveats** — the chain walk enforced scope-subset + temporal
   attenuation but NOT `delegation_caveats` (`chain_no_delegation_denied`,
   `chain_max_delegation_ttl_denied`). Added `check-delegation-caveats` per-link in
   `verify-capability-chain`, plus the §5.5a per-link granter frames in
   `scope-subset`/`grant-subset`/`is-attenuated` (hard-fail deny on an unresolvable
   link granter).

Second run: **568 / 284P / 195W / 0 FAIL / 89S → PASS.**
