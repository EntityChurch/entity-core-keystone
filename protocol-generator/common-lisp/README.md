## What it is

A pure-Common-Lisp peer whose **only third-party runtime dependency is `ironclad`** (crypto):
the canonical-CBOR (ECF) codec, Ed25519/Ed448 identity + deterministic signatures, capability
authorization with chain attenuation, the §4.1 handshake / §6.5 dispatch machinery, the §9.5
53-type registry, and the address-space store — no FFI, no C bindings, no libsodium.

- **Codec:** hand-rolled canonical CBOR (`src/cbor.lisp`) — shortest-float minimization
  (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip), length-then-lex map-key sort on encoded
  key bytes, recursive major-type-6 tag rejection on decode. LEB128 varints (`src/varint.lisp`)
  + Base58 (`src/base58.lisp`) hand-rolled (no CL CBOR library gives ECF's canonical guarantees).
- **Crypto (the contrast with OCaml):** Ed25519 **and** Ed448 are both **native, pure-Lisp** via
  `ironclad` — no FFI, no opt-in agility sub-library. Where OCaml had to source Ed448 over a
  C-ABI (A-OC-002) and Zig has no Ed448 at all (A-ZIG-002), the CL peer reaches the §9.1 floor
  (Ed25519 + SHA-256) **and** the agility higher bar (Ed448 + SHA-384) from the default build.
  Ed448 byte-equality is RFC-8032-KAT-gated (A-CL-005). SHA-256/384 via `ironclad:digest-sequence`.
- **Integers:** Common Lisp has **native arbitrary-precision integers**, so the full §3.2
  0..2⁶⁴−1 carrier range is represented with no workaround — where OCaml hit a 63-bit trap
  (A-OC-001) and TS escalated to bigint (F7), CL is correct-by-construction with stock bignums.
- **Error model (the condition system):** decode rejections + protocol errors are CL
  **conditions** (`entity-core-error` and subtypes — `non-canonical-ecf`, `truncated-input`,
  `tag-rejected`, `unsupported-content-hash-format`, …); restarts are available but the dispatch
  brain carries protocol status as a value record, never across a condition (the cohort invariant).
- **Dispatch (the distant-idiom probe, A-CL-008):** §6.5/§6.6 operation dispatch is a CLOS
  GENERIC FUNCTION with **multiple dispatch** on `(handler-class × operation)` — each MUST handler
  is a class, each operation an EQL-specialized method on `HANDLE-OP`, and "unknown operation → 501"
  is the default method. The four prior peers express the same §6.6 surface as a single-dispatch
  match ladder; the convergence on byte/behavior-identical dispatch is an idiom-neutrality signal.
- **Concurrency:** SBCL native threads (`sb-thread`, no third-party dep) — one reader thread per
  connection demuxing `EXECUTE_RESPONSE` by `request_id`, inbound EXECUTE dispatched on its own
  thread so it never blocks outbound (§4.8 / §6.11). `bordeaux-threads` deferred (A-CL-003).

## Build / test / run-conformance (in-container, offline)

All build/test/conformance runs happen in the pinned `common-lisp-toolchain` Podman image
(`fedora:43` base + source-built **SBCL 2.6.4**, SHA-256-pinned; **ironclad 0.61** pre-installed
at container-build time from a pinned Quicklisp dist into `/opt/quicklisp`).
After that layer the image needs no network: every run is fully sealed-offline (`--network=none`),
ASDF resolves `ironclad` from disk. There is **no host SBCL dependency**.

```sh
# S2 codec conformance (69/69 byte-identical, sealed offline) + Ed448 RFC-8032 KAT
./run-s2.sh              # full gate (conformance + selftest)
./run-s2.sh conform      # ECF corpus PASS/FAIL counts only

# S3 two-peer loopback smoke (handshake + dispatch + capability + 8-way request_id demux)
./run-s3.sh

# S4 live-peer conformance (validate-peer --profile core). The Go oracle ELF and the
# SBCL host run together inside the common-lisp-toolchain image, sharing one loopback, offline.
./run-s4.sh

# Origination-core reentry probe (§6.11 / §10.2) — CL target + Go entity-peer reference, 3/3
./run-origination-core.sh
```

The S4/origination oracles (`output/s4-oracles/{validate-peer,entity-peer}`) are `CGO_ENABLED=0`
Go ELFs built from `entity-core-go` HEAD; rebuild them per the runbook in
[`status/CONFORMANCE-REPORT.md`](status/CONFORMANCE-REPORT.md) §Reproduce.

## Install / use

The peer is an **ASDF system** (three systems, layered):

- `entity-core` — the codec island (package `ENTITY-CORE`, nickname `EC`). Deps: `ironclad`.
- `entity-core/peer` — the full peer (package `ENTITY-CORE/PEER`, nickname `ECP`). Deps:
  `entity-core`, `ironclad`, `sb-bsd-sockets`.
- `entity-core/test` — the hand-rolled codec conformance harness (test-op; not a public surface).

```lisp
;; Make the system discoverable, then load it.
(push (truename "/path/to/protocol-generator/common-lisp/") asdf:*central-registry*)
(asdf:load-system :entity-core/peer)

;; ── Tier 1 — codec island: encode/verify ECF (no peer) ──
(let* ((e     (ecp:make-entity :typ "system/peer"
                               :data (ec:make-cbor-map :pairs '(("name" . "alice")))))
       (bytes (ec:cbor-encode (ecp:entity-to-cbor e)))   ; canonical ECF bytes
       (id    (ecp:identity-of-seed seed)))              ; Ed25519 identity + peer_id
  (declare (ignore bytes id)))

;; ── Tier 2 — full peer ──
(let ((peer (ecp:make-peer :seed seed)))
  (ecp:start-listener peer :port 7777))
```

Public surface is tiered (see [`status/PHASE-S5.md`](status/PHASE-S5.md) §3 and the export block
in `src/peer-package.lisp`): Tier 1 = model + identity, Tier 2 = full peer. The test-client +
address-space helpers (`empty-params`, `resource-target`, `scope`, `hex`, …) are exported for the
in-repo test execs but are **not** a stable surface and may churn without a semver bump.

## Distribution (when published)

The Common Lisp community channels are **Quicklisp** (the de-facto dist) and **Ultralisp** — both
index *git repositories*, so "publishing" is registering the repo; there is no upload step. The
shipped artifact is a self-contained ASDF system + README + LICENSE. ASDF's `:version` field is
dotted-integer only (it rejects a SemVer `-pre` suffix), so the `:version` slots carry the
parseable `0.1.0` while the `0.1.0-pre` pre-release LINE lives in this README + `CHANGELOG.md`
(A-CL-010). See [`status/PHASE-S5.md`](status/PHASE-S5.md) §Operator-handoff and
[`status/ARCHITECTURE-REVIEW.md`](status/ARCHITECTURE-REVIEW.md) §Publishing-options for the
decision surface (operator-decides; not acted on by the pipeline).
