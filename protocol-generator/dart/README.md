# entity-core-protocol-dart

[`entity-core-keystone`](../../README.md) `/entity-rosetta` pipeline. A **release "reach" peer** —
built for **Flutter cross-platform / Dart-ecosystem coverage** (V7 Layers 0–4 core), not as a
spec-discovery instrument: the closest idiom analogues (Kotlin's sealed-class `Result`, TypeScript's
wide-integer carrier for the uint64 head-form) were already exercised by the cohort, so Dart is
**corroboration-only** on the spec axis (anything it surfaces anyway is high-signal and gets logged +

(NOT FFI into the sibling C-ABI codec, NOT the `cbor` pub package; A-DART-001), so where a value
matches another peer it is by independent arrival from the shared spec, not by inheritance.

## What it is

A pure-Dart peer whose **only non-first-party runtime dependency is `cryptography_plus`** (Ed25519):
the canonical-CBOR (ECF) codec, Base58, the multicodec LEB128 varint, content-hash, peer-id, the §4.1
handshake / §6.5–§6.6 dispatch machinery, capability authorization with chain attenuation + §5.7
delegation caveats, the §9.5 53-type registry, the §9.5a CORE-TREE get/put/CAS/delete surface, the
§10.1 core-register gate, the §4.10 resource-bounds checks, and the §7b concurrency gate are all
hand-rolled in `lib/src/`. Being pure Dart, the peer is **self-contained on every Flutter target
(iOS / Android / web / desktop) with no native library to ship** — the headline reach value.

- **Codec:** hand-rolled canonical CBOR (`lib/src/codec/ecf.dart`, on `Uint8List`/`ByteData`) —
  shortest-float minimization (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip), RFC-7049
  **length-first-then-bytewise (CTAP2)** map-key ordering (≠ RFC-8949 §4.2 bytewise — the trap stock
  CBOR libs fall into), recursive major-type-6 **tag rejection at any depth** → `400 non_canonical_ecf`,
  definite-length only. No `cbor` pub package (none gives ECF canonicality — A-DART-001). Base58 +
  LEB128 varint hand-rolled (neither is in the Dart core libs).
- **uint64 head-form carrier = `BigInt`** (the headline codec call; A-DART-006). Dart `int` is 64-bit
  on the native VM but **53-bit on web / dart2js** — `BigInt` carries the full `0 .. 2⁶⁴−1` head-form
  range **web-safe AND native-correct**. This is the TypeScript-`bigint` / web-number lesson, Dart
  edition; it is the single most load-bearing portability decision. The `dart2js` web-truncation proof
  is run as a test (A-DART-013). See **Deferred / known limitations** below for the consumer cost.
- **Crypto:** Ed25519 sign/verify + the §1.5 raw-pubkey via **`cryptography_plus` 2.7.1** (the
  *maintained* community fork — the original `cryptography` is stalled; A-DART-002/012); SHA-256 via
  **`package:crypto` 3.0.6** (first-party Dart-team). Both pure Dart → browser + native. RFC-8032
  deterministic signing → cross-impl signature byte-equality. **Ed448 / SHA-384 agility is deferred**
  (A-DART-003) — there is no maintained pure-Dart Ed448, and an FFI route would break the pure-Dart
  self-containment that is this peer's whole point; the §9.1 floor (Ed25519 + SHA-256) is fully native
  and is the only path the corpus exercises.
- **Identity / peer_id:** §1.5 canonical-form construction (Ed25519 → `hash_type=0x00`
  identity-multihash, the digest IS the raw 32-byte pubkey), per the §1.5 v7.65 table, **not** the
  stale §7.4 SHA-256 pseudocode (corroboration; A-DART-010). The seed-`0x11`×32 peer_id
  `2KHoAk7A5JmhygZJAdBua8iRD1CnBoJRfUBHgZeXNRTeFg` is byte-identical to the Kotlin / Java / Common
  Lisp peers. Lowercase `%02x` hex everywhere.
- **Error model:** Dart-3 **sealed-class `Result`** matched by **exhaustive switch expression**
  (compiler-enforced — the static-rigor analogue of Kotlin's `when` / OCaml's `result`). Protocol
  status is carried as a value, never across an exception (the cohort's status-as-value invariant);
  exceptions are reserved for programmer-error, caught at the §4.9 connection boundary. (A-DART-004.)
- **Concurrency:** `Future`/`async-await` on the per-isolate **event loop**. §7b store-safety is
  **structural** — a single-threaded event loop means no lock is needed in-isolate (A-DART-005). The
  S4 concurrency gate is 5/5; the one informational WARN is "no parallel speedup on a single-threaded
  loop" (not a §6.11 violation — the §6.11(a) no-serialization MUST is enforced by `t1_3`, which
  passes). The transport carries an O(1)-cursor frame reassembler + a connection-set lifecycle
  (A-DART-016, the one genuine peer bug S4 surfaced and fixed — an I/O-path complexity bug, not a spec
  defect).

## Use

Two import tiers (two `library`-level exports):

```dart
import 'package:entity_core_protocol/entity_core_protocol.dart'; // Tier 1 — codec + crypto island
import 'package:entity_core_protocol/entity_core_peer.dart';     // Tier 2 — full peer + transport

// Tier 1 — encode/verify ECF (no peer)
final Uint8List bytes = encodeEcf(entity.toCbor());   // canonical ECF bytes
final id = peerIdFromSeed(seed);                       // Ed25519 identity + §1.5 peer_id

// Tier 2 — full peer over real TCP
final peer = Peer(/* identity, grant policy */);
await startListener(peer, port: 7777);                 // Future per connection on the event loop
```

Internal units (`lib/src/...` — varint, Base58, wire framing, the type-registry render table) are
implementation detail behind the two top-level libraries and may churn without a semver bump. An
explicit public-surface freeze is a mechanical publish-prep pass, deferred until the surface is frozen
against a first external consumer — the honest `0.1.0-pre` state (mirrors the Kotlin / Java / OCaml /
Zig deferral).

## Deferred / known limitations

- **Ed448 / SHA-384 crypto-agility is deferred** (A-DART-003) — no maintained pure-Dart Ed448, and an
  FFI route would break the pure-Dart self-containment. Does NOT affect the §9.1 floor (Ed25519 +
  SHA-256, 69/69 byte-green) nor the connect path; only the agility MATRIX harness is unwired. (The
  SHA-384 *hash* is available in `cryptography_plus` should the higher bar be taken later.)
- **Web / dart2js `BigInt` cost** (A-DART-006/013) — the uint64 head-form rides `BigInt` to survive the
  53-bit web-int truncation. The correctness win is the whole point on the web target; the cost is
  `BigInt` arithmetic on the hot codec path (heavier than native `int`). For native-VM-only consumers
  this is pure overhead, but the reach mandate (Flutter web included) makes the web-safe carrier the
  right floor. The truncation behavior is locked by a `dart2js`-runtime test.
- **Public API surface is documented (the two Tier libraries), not yet frozen** with an explicit
  visibility lock — deferred to publish-prep / first external consumer.
- **pub.dev publishing is deferred** (A-DART-007) — requires a verified pub.dev *publisher namespace*
  before the first `dart pub publish`. The package is publish-ready (`0.1.0-pre`, `dart pub publish
  --dry-run` green); the deploy is an operator action. See [`packaging/README.md`](packaging/README.md)
  and [`status/PHASE-S5.md`](status/PHASE-S5.md) §Operator-handoff.
- The v7.73/v7.74/v7.75 peer-surface behavior is oracle-sourced against the v7.75 spec-data snapshot
  (the v7.76/v7.77 deltas are extension-only; the core floor is byte-unchanged).

## Distribution (when published)

Dart's registry is **pub.dev**. Publishing is the stock `dart pub publish`, gated on a **verified
pub.dev publisher namespace** (e.g. `entitycore.org`) — a one-time operator step claimed at first
publish (A-DART-007). The `pubspec.yaml` carries the `0.1.0-pre` version, the Apache-2.0 license, the
pinned deps, and `repository`/`homepage`/`topics` metadata; `publish_to: none` is the parked-state
guard against accidental deploy (it is ignored by `--dry-run`, which still runs the full pub.dev
validation suite). "Publishing" is removing that line + `dart pub publish` against the verified
namespace. See [`packaging/README.md`](packaging/README.md).
