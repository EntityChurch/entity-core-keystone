## What it is

A native Ada 2012/2022 (GNAT) peer with a **single runtime dependency** — system libsodium (a
universal, audited shared library) via a thin `Interfaces.C` binding. The canonical-CBOR (ECF)
codec, the §4.1 handshake / §6.5–§6.6 dispatch machinery, capability authorization with chain
attenuation, the §9.5 53-type registry, the §9.5a CORE-TREE get/put/CAS/delete surface, the §6.11
reentry seam, and the §4.10 resource-bounds floor — no Alire crate dependencies (CBOR + Base58 +
varint hand-rolled). This ties Zig/Elixir/Java for the lightest supply-chain in the cohort, here via
the Fedora 43 distro toolchain + a system crypto library.

- **Codec:** hand-rolled canonical CBOR (`Entity_Core.Codec.Cbor` / `.Value`) — shortest-float
  minimization (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip), length-then-lex map-key sort on
  encoded key bytes, recursive major-type-6 tag rejection on decode, definite lengths. **Ada
  advantage — no unsigned trap:** Ada has native modular integer types (`Interfaces.Unsigned_64`),
  so the full §3.2 0..2⁶⁴−1 head-form range is native — unlike the C# `ulong` / Java `long`
  no-native-unsigned workaround. Wire bytes are `Stream_Element_Array` / `Unsigned_8`, never
  `String` (no char-encoding ambiguity). Base58 + the multicodec LEB128 varint are hand-rolled
  (neither exists in a stock Ada library).
- **Crypto:** Ed25519 sign/verify + SHA-256 via the **libsodium Interfaces.C binding**
  (`Entity_Core.Crypto`: `crypto_sign_seed_keypair` / `crypto_sign_detached` /
  `crypto_sign_verify_detached` / `crypto_hash_sha256`), deterministic RFC-8032. **Ada advantage
  over the Java peer's crypto wrinkle:** libsodium returns the **raw 32-byte public key directly**
  — no point-encoding extraction step (contrast the JDK EdEC `(sign-bit, y-coordinate)` decode).
  Ed448 + SHA-384 (the v7.67 agility higher bar) are **deferred** — libsodium has neither; the
  §9.1 core floor needs only Ed25519 + SHA-256 (A-ADA-002).
- **Identity / peer_id:** §1.5 canonical-form construction (Ed25519 → `hash_type=0x00`
  identity-multihash, digest = the raw pubkey), following the §1.5 table — **not** the stale §7.4
  `SHA256(pubkey)` form (A-ADA-001). Lowercase `a-f` hex everywhere via a custom nibble→char table
  — Ada's hex builtins (`Integer'Image`, `16#..#`, `Integer_IO Base=>16`) default UPPERCASE, which
  would 404 the case-sensitive §3.4/§3.5/§6.9a tree-path keys against the lowercase oracle (the
  A-CL-009 trap, pinned proactively as A-ADA-003). Peer ID `2KD6sD8JpEHJ3EaQu2mKCfiQZnkvcDmS8xtvstw9c4dHZm`.
- **Data model:** §1.1 entity `data` is an arbitrary ECF value, modeled as a **discriminated-record
  variant** over the full ECF value space — **not** an `Ada.Containers` map (A-ADA-009 / A-JAVA-010).
  Ada's strong, static typing makes this load-bearing: a map-only field is a type-level commitment
  the compiler accepts and only the live oracle catches (the silent-500 trap on the first scalar-data
  entity).
- **Error model (the design-by-contract seam):** Ada **exceptions** (a typed lattice rooted at
  `Entity_Core_Error` — Codec_Error / Crypto_Error / Protocol_Error / Transport_Error subtypes)
  **paired** with **design-by-contract**: Pre/Post/Type_Invariant aspects (Ada 2012+) guard internal
  invariants (a Content_Hash is exactly 32 bytes; a canonical buffer is well-formed; a peer_id is a
  valid multihash) — runtime-checked under `-gnata`, the static-rigor seam no prior peer exercised.
  Protocol status is mapped from an exception subtype → status code at the dispatcher boundary
  (`Authentication_Error`→401, `Authorization_Error`→403, `Chain_Depth_Exceeded_Error`→**400** per
  §4.10(b), `Payload_Too_Large_Error`→413). SPARK formal proof is out of scope for v0.1 (the aspects
  are runtime guards, not discharged proofs).
- **Concurrency (the centerpiece seam):** Ada's first-class **tasks + protected objects +
  rendezvous** — a model no prior peer used (the cohort spanned BEAM actors, STM, green threads, raw
  OS threads, JVM virtual threads, goroutines, cooperative async). The §4.8 store + tree index live
  inside a **protected object**, so the store-race is **structurally unrepresentable** (the C
  sibling's heap race A-C-009 and the Zig/CL manual store-safety cannot occur here by construction —
  the cleanest §4.8 story in the cohort). One **task per connection** (GNAT maps tasks to OS threads,
  so a blocking read never stalls other connections); the §6.11/N7 demux and the shared-stream write
  serialization are themselves protected objects; TCP_NODELAY on accepted sockets. All 5 §7b checks
  PASS, genuinely concurrent (5/5 under 10000-req sustained load, not serialized).

## Use

The public surface is the `Entity_Core.*` library hierarchy, built by the `.gpr` into the
`Entity_Core_Protocol` library. Two tiers:

```ada
with Entity_Core.Codec.Cbor;      -- Tier 1 — codec island (canonical ECF encode/decode)
with Entity_Core.Codec.Peer_Id;   -- Tier 1 — §1.5 identity-multihash peer_id
with Entity_Core.Crypto;          -- Tier 1 — Ed25519 sign/verify + SHA-256 (libsodium binding)
with Entity_Core.Protocol;        -- Tier 2 — the full peer (handshake, dispatch, store, capability)
```

Consume by adding the `.gpr` as a `with`'d project (`with "entity_core_protocol.gpr";` in your own
GNAT project) or by building the library and linking it (`-lsodium` is the one runtime link). The
runtime needs system **libsodium 1.0.22** present (`libsodium` Fedora package, or the equivalent on
your distro). Internal units (varint, base58, wire framing, the type-registry render table) are
implementation detail and may churn without a semver bump. An explicit public-surface freeze is a
mechanical publish-prep pass, deferred until the surface is frozen against a first external consumer
— the honest `0.1.0-pre` state (mirrors the Java public-surface / OCaml `.mli` deferral).
