## What it is

A pure-Rust peer with a **deliberately minimal dependency closure** ŌĆö **two direct registry
deps** (`ed25519-dalek` + `sha2`); the ECF canonical-CBOR layer, Base58, and the LEB128 varint
are all hand-rolled (no `ciborium` / `bs58` / `leb128`). `#![forbid(unsafe_code)]` on the
crate. `cargo build` / `cargo test` pull nothing off the network once `Cargo.lock` is vendored.

- **Codec:** hand-rolled canonical CBOR (`src/cbor.rs`) ŌĆö Rule 1 (shortest int head), Rule 2
  (length-then-lex map-key sort on the *encoded* key bytes ŌĆö equivalent to bytewise-on-encoded
  for the ECF key space; see A-RUST-005), Rule 3 (definite lengths), Rule 4/4a (shortest-float
  ladder f64ŌåÆf32ŌåÆf16 with decode-side minimality enforcement), Rule 5 (no duplicate keys),
  ┬¦6.3 (recursive major-type-6 tag rejection on decode ŌåÆ `400 non_canonical_ecf`). LEB128
  multicodec varints (`src/varint.rs`) + Base58 (`src/base58.rs`), both hand-rolled (neither is
  in `std`). No surveyed Rust CBOR lib gives ECF's guarantees for free (A-RUST-001).
- **Crypto:** Ed25519 via `ed25519-dalek` (`=2.2.0`, audited, RFC-8032 deterministic signing ŌĆö
  no RNG to sign); SHA-256 (content_hash floor) + SHA-384 (agility hashing) via `sha2`
  (`=0.10.9`), both native. *Ed448 / the crypto-agility higher bar is **not yet supported** ŌĆö
  native-full-agility incl. Ed448 is not cleanly reachable in Rust (RustCrypto `ed448-goldilocks`
  has signing only in a `0.14.0-pre` prerelease with an unaudited caveat; the stable `0.9.0` has
  no signing API). The planned path is **hybrid FFI**: native Ed25519 (shipped) + FFI Ed448 via
  `libentitycore_codec`'s `ec_ed448_*` over the C-ABI when agility enters scope ŌĆö Rust sits in the
  cohort's gapŌåÆhybrid-FFI band with Go / Zig / OCaml / Swift. Does **not** affect the ┬¦9.1
  Ed25519+SHA-256 conformance floor (Ed448 is validated, not required). See
  [`status/SPEC-AMBIGUITY-LOG.md`](status/SPEC-AMBIGUITY-LOG.md) A-RUST-002 (the headline crypto
  finding) and A-RUST-004.*
- **Integers:** CBOR head-form carrier over native `u64`/`i64` ŌĆö the full ┬¦3.2
  0..2ŌüČŌü┤ŌłÆ1 / ŌłÆ1..ŌłÆ2ŌüČŌü┤ range maps directly, no BigInt ceremony and no 63-bit trap (the
  `[-2ŌüČŌü┤,-1]` `nint` band is carried with explicit `u64` arithmetic).
- **Concurrency:** `std::thread` + `std::sync` (no `tokio` ŌĆö dep-minimization, A-RUST-003). One
  reader thread per connection demuxing `EXECUTE_RESPONSE` by `request_id`; each inbound EXECUTE
  on its own thread so it never blocks outbound (┬¦6.11). The ┬¦4.8 content store is an
  `RwLock<HashMap>` ŌĆö an unsynchronized shared-mutable store is a *compile error* in Rust, so the
  Zig/Common-Lisp store-race is structurally unrepresentable here. Validated live by the oracle's
  `concurrency` category (5/5 PASS, incl. the ┬¦7b T2.1 sustained-load store-race probe).

Podman image (`fedora:43` + distro `rust`/`cargo` `1.96.0-1.fc43`, a reviewed channel; clippy +

## The host binary (`entity-peer-host`)

The peer ships a runnable host (the S4 conformance driver and a general single-peer listener):

```
entity-peer-host --port N [--name NAME] [--validate] [--debug-open-grants] [--help]
```

- `--name NAME` ŌĆö load (or provision) the peer's persistent identity at
  `~/.entity/peers/NAME/keypair` (the Go `entity-peer` / peer-manager convention); the seed
  fixes a deterministic `peer_id`. Emits a `LISTENING ŌĆ”` line once the listener is up.
- `--validate` ŌĆö enable the ┬¦7a `system/validate/{echo,dispatch-outbound}` conformance handlers
  (so the validator's probes run live instead of honest-SKIP).
- `--debug-open-grants` ŌĆö the degenerate open seed-policy the grant-gated probe categories need.

There is also `wire-conformance --input <corpus.cbor> [--out ŌĆ”] [--json ŌĆ”]` (the Go
`emit-canonical` analogue) for the S2 codec gate.

## Use (library surface)

The public API is the crate `entity_core_protocol` (`[lib] name`). Two tiers:

```rust
use entity_core_protocol::{cbor, content_hash, peer_id, signature, value::Value};

// Tier 1 ŌĆö codec island: encode/verify ECF, content-hash, peer-id, sign/verify
let bytes = cbor::encode(&value)?;                  // canonical ECF bytes (Result idiom)
let h     = content_hash::content_hash(&bytes);     // 0x00 || SHA-256(ECF)
// Tier 2 ŌĆö full peer
use entity_core_protocol::peer;
let p = peer::Peer::new(/* seed, config */)?;       // boot a peer, then serve over TCP
```

Exact entry points carry `///` rustdoc on the exported surfaces (`cargo doc`). The codec
internals (`cbor`, `base58`, `varint`) are `pub` for the conformance harness but are
implementation detail and may churn without a semver bump. The `peer` module's exported surface
is documented but not yet frozen against a semver lock ŌĆö deferred to publish-prep / first
external consumer (see [`status/PHASE-S5.md`](status/PHASE-S5.md) ┬¦Public-surface).
