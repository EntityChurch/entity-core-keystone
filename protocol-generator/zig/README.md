# entity-core-protocol-zig

A native-Zig **core protocol peer** for the Entity Core protocol (V7), generated by the
[`entity-core-keystone`](../../README.md) `/entity-rosetta` pipeline. Peer **#4**, derived
**spec-first** (no sibling-language source) and carried into a *distant idiom* — a systems
language with **no GC, explicit allocators, error unions, and `comptime`**.

**Conformance:** `validate-peer --profile core` — **PASS** · 568 total · 284 pass · 195 warn ·
**0 fail** · 89 skip (machine-verified `summary.failed == 0`). Codec: 69/69 byte-identical to
`conformance-vectors-v1`, first run, 0 codec fixes. The 568 total (vs the C#/TS-era 552) is the
v7.74 oracle's 16 extra checks (§10.1 core-register round-trip + §9.5a CORE-TREE vectors).
See [`status/CONFORMANCE-REPORT.md`](status/CONFORMANCE-REPORT.md).

**Status:** `0.1.0-pre` — publish-ready, **not yet published** (pending arch v0.1 sign-off +
first external consumer). Tracks V7 spec-data **v7.72 + v7.74** closeout; codec corpus v0.8.0.

## What it is

A pure-Zig peer (**zero third-party packages — `std`-only**): canonical-CBOR (ECF) codec,
Ed25519 identity + deterministic signatures, capability authorization with chain attenuation,
the §4.1 handshake / §6.5 dispatch machinery, the §9.5 53-type registry, and the §9.5a
CORE-TREE get/put/CAS/delete surface — no FFI, no C bindings, no libsodium. `zig build` pulls
nothing off the network.

- **Codec:** hand-rolled canonical CBOR (`src/cbor.zig`) — shortest-float minimization
  (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip), length-then-lex map-key sort on encoded
  key bytes, recursive major-type-6 tag rejection on decode, full `uint`/`nint` 0..2⁶⁴−1 range.
  `comptime` dispatches the encoder on type with zero runtime reflection. LEB128 varints
  (`src/varint.zig`) + Base58 (`src/base58.zig`) hand-rolled (neither is in `std`).
- **Crypto:** Ed25519 via `std.crypto.sign.Ed25519` (in `std`, audited, RFC-8032 deterministic
  signing — no RNG for signing); SHA-256/384 via `std.crypto.hash.sha2`. Cross-impl signature
  byte-equality. *Ed448 / crypto-agility higher bar is not yet supported — `std.crypto` has no
  Ed448 and no audited pure-Zig Ed448 exists; the planned path is hybrid (native Ed25519 + FFI
  Ed448). See [`status/SPEC-AMBIGUITY-LOG.md`](status/SPEC-AMBIGUITY-LOG.md) (A-ZIG-002).*
- **Integers:** CBOR head-form carrier — full 0..2⁶⁴−1 over native `u64`, builds in
  ReleaseSafe so varint/length overflow traps loudly rather than wrapping (A-ZIG-001 §note).
- **Memory (the headline Zig seam):** no GC. Every allocating API takes an
  `allocator: std.mem.Allocator`; decode follows a documented caller-frees ownership contract;
  the peer dispatch path runs each request against a per-request `ArenaAllocator` and
  deep-clones the response into the long-lived allocator (A-ZIG-007). `std.testing.allocator`
  fails any test that leaks — **free-correctness is a first-class conformance concern here**,
  unlike the GC'd C#/TS/OCaml peers.
- **Concurrency:** OS threads (`std.Thread`, in `std`) — one reader thread per connection
  demuxing `EXECUTE_RESPONSE` by `request_id` (§4.8 / §6.11). Zig's `async` is in flux
  ("Writergate"); threads are the stable, zero-dep choice (A-ZIG-003).

## Build / test / run-conformance (in-container, offline)

All build/test/conformance runs happen in the pinned `zig-toolchain` Podman image
(`fedora:43` base + the official ziglang.org **0.15.1** tarball, SHA-256-pinned + minisign
signature-verified). The peer is `std`-only, so every run is fully sealed-offline
(`--network=none`). There is **no host Zig dependency**.

```sh
# Build the static codec library (+ host/smoke/conformance executables)
podman run --memory=4g --memory-swap=4g --pids-limit=2048 --cpus=4 --rm --network=none -v "$PWD":/work:Z -w /work/protocol-generator/zig \
  entity-core-keystone/zig-toolchain:latest zig build

# Unit tests — leak-checked (std.testing.allocator); includes the S2 69/69 codec
# vectors + the A-ZIG-008 53-type byte-diff + deletion-marker + §7a echo tests
podman run --memory=4g --memory-swap=4g --pids-limit=2048 --cpus=4 --rm --network=none -v "$PWD":/work:Z -w /work/protocol-generator/zig \
  entity-core-keystone/zig-toolchain:latest zig build test

# S2 codec wire-conformance (69/69 byte-identical, sealed offline)
podman run --memory=4g --memory-swap=4g --pids-limit=2048 --cpus=4 --rm --network=none -v "$PWD":/work:Z -w /work/protocol-generator/zig \
  entity-core-keystone/zig-toolchain:latest \
  zig build conformance -- \
    /work/protocol-generator/shared/test-vectors/v0.8.0/conformance-vectors-v1.cbor

# S4 live-peer conformance (validate-peer --profile core). The Go oracle ELF and the
# Zig peer run together inside the zig-toolchain image, sharing one loopback, offline.
podman run --memory=4g --memory-swap=4g --pids-limit=2048 --cpus=4 --rm --network=none -v "$PWD":/work:Z \
  entity-core-keystone/zig-toolchain:latest sh /work/protocol-generator/zig/run-s4.sh
```

The S4 oracle (`output/s4-oracles/validate-peer`) is a `CGO_ENABLED=0` Go ELF built from
`entity-core-go` HEAD; rebuild it per the runbook in
[`status/CONFORMANCE-REPORT.md`](status/CONFORMANCE-REPORT.md) §Reproduce when the oracle moves.

## Use

Public module surface is `src/root.zig` (Zig module `entitycore_codec`). Two tiers:

```zig
const ec = @import("entitycore_codec"); // build.zig: root_source_file = src/root.zig

// Tier 1 — codec island: encode/verify ECF (no peer)
var entity = try ec.Model.Entity.init(gpa, "system/peer", data_map); // data_map: ec.cbor.Value
defer entity.deinit(gpa);
const bytes = try ec.cbor.encode(gpa, entity.toCbor());               // canonical ECF bytes
defer gpa.free(bytes);
const id = try ec.Identity.ofSeed(seed);                             // Ed25519 identity + peer_id

// Tier 2 — full peer
var peer = try ec.Peer.create(gpa, .{ .seed = seed, .open_grants = false });
defer peer.deinit();
try ec.transport.acceptLoop(&peer, 7777);
```

Exact entry points are re-exported from `src/root.zig`; `defer`/`errdefer` ownership is
documented per module (`model.zig`, `store.zig`). Internal units (`varint`, `base58`, `wire`,
`type_defs`) are implementation detail and may churn without a semver bump.

## Distribution (when published)

Zig has **no central package registry**. "Publishing" is a git tag that consumers pin by URL +
content hash in their own `build.zig.zon` — decentralized and hash-pinned by design (itself a
supply-chain-friendly property). No `entity_core_protocol_zig` is fetched off any index. See
[`status/PHASE-S5.md`](status/PHASE-S5.md) §Operator-handoff and
[`status/ARCHITECTURE-REVIEW.md`](status/ARCHITECTURE-REVIEW.md) §Publishing-options for the
in-repo-vs-standalone-repo decision surface (operator-decides; not acted on by the pipeline).
