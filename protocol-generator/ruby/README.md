# entity-core-protocol-ruby

A native-Ruby **core protocol peer** for the Entity Core protocol (V7), generated by the
[`entity-core-keystone`](../../README.md) `/entity-rosetta` pipeline. Peer **#12** — the cohort's
**first dynamic / duck-typed / scripting peer** and the **10th byte-compatible core impl** (after
the Go/Rust/Python reference trio + C#/TS/OCaml/Elixir/Common-Lisp/Zig/Swift/Haskell/Java) — derived
**spec-first** (no sibling-language source) in idiomatic MRI/CRuby: open classes, blocks, mixins, a
`StandardError`-rooted exception hierarchy, `Data.define` immutable value objects, and GVL-aware
thread-per-connection concurrency.

**Conformance:** `validate-peer --profile core` — **PASS** · 653 total · 291 pass · 268 warn ·
**0 fail** · 94 skip (machine-verified `summary.failed == 0`), oracle `entity-core-go @75c532e`.
All 16 core-profile categories are 0-FAIL. Codec: 69/69 byte-identical to `conformance-vectors-v1`,
first full run, 0 codec fixes. (The 653 total vs the v7.75 8-peer-rerun's 576 is purely later-oracle
**extension** categories — `relay`/`discovery`/`registry`/`published_root` — that auto-skip under
`--profile core`; the FAIL gate and every core category are unchanged. See
[`status/CONFORMANCE-REPORT.md`](status/CONFORMANCE-REPORT.md).) Plus origination-core 3/3 over real
two-peer TCP and the §9.5 53-type floor 53/53 byte-identical.

**Status:** `0.1.0-pre` (RubyGems coordinate `0.1.0.pre` — see *Versioning* below) — publish-ready,
**not yet published** (pending arch v0.1 sign-off + first external consumer). Tracks V7 spec-data
**v7.75** (the complete ratified snapshot — no snapshot-lag caveat); codec corpus byte-identical
v7.73→v7.75.

## What it is

A native Ruby peer with a **zero-runtime-gem-dependency core**: a canonical-CBOR (ECF) codec,
Ed25519 **and Ed448** identity + deterministic signatures, capability authorization with chain
attenuation + §5.7 delegation caveats, the §4.1 handshake / §6.5–§6.6 dispatch machinery, the §9.5
53-type registry, the §9.5a CORE-TREE get/put/CAS/delete surface, the §4.10 resource_bounds floor,
and the v7.73/v7.74 peer surface (§6.13 register, §PR-8 granter frame, §6.9a owner-cap, §7a
conformance handlers). No FFI; `bundle install` pulls **nothing** onto the runtime load path.

- **Codec:** hand-rolled canonical CBOR (`EntityCore::Cbor`) — shortest-float minimization
  (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip; `Array#pack` has no half-float code so f16 is
  assembled from the binary64 bits), length-then-lex map-key sort on encoded key bytes, recursive
  major-type-6 tag rejection on decode. The CBOR head-form integer carrier is just a Ruby
  **`Integer`** — arbitrary-precision, so the full 0..2⁶⁴−1 range carries with **no native-int
  trap** (the BEAM result, replicated; no `ulong`/int63/`bigint` workaround). Base58
  (`EntityCore::Base58`) + the multicodec LEB128 varint (`EntityCore::Varint`) are hand-rolled.
  Wire bytes are **ASCII-8BIT** (`String#b`) throughout — never UTF-8 for the codec core.
- **Crypto:** Ed25519 + Ed448 sign/verify + SHA-256/384 via the **stdlib `openssl`** gem (OpenSSL
  3.x, bundled with Ruby) — zero-dep, native, **no FFI, no libsodium/RbNaCl**. The generic PKey
  EdDSA surface reaches *both* curve families, making Ruby the **third native-full-agility
  substrate** (after Elixir `:crypto` and Haskell crypton, and the first via OpenSSL stdlib).
  Byte-verified against the v7.67 agility pins. See
  [`status/SPEC-AMBIGUITY-LOG.md`](status/SPEC-AMBIGUITY-LOG.md) (A-RUBY-002, A-RUBY-003).
- **Identity / peer_id:** §1.5 canonical-form construction (`hash_type=0x00` identity-multihash,
  digest = raw pubkey for ≤32 B keys), following the §1.5 v7.65 table, **not** the stale §7.4
  SHA-256 pseudocode. Lowercase hex everywhere (dodges the A-CL-009 uppercase address-space-path
  trap that bit Common Lisp).
- **Error model:** a `StandardError`-rooted `EntityCore::Error` exception lattice (`CodecError`,
  `ProtocolError`, `TransportError`, …) — the canonical Ruby fallible idiom (`raise`/`rescue`),
  distinct from Elixir's tagged tuples / OCaml's `result` / Zig's error unions. Protocol status is
  carried as a **value record, never across an exception** (the cohort's status-as-value invariant);
  §5.2/§5.2a verification returns a three-way verdict (ALLOW / unauthenticated→401 /
  authenticated-but-unauthorized→403 / unresolvable-identity→401).
- **Concurrency (the GVL seam):** **thread-per-connection** under MRI's Global VM Lock — one reader
  `Thread` per connection demuxing `EXECUTE_RESPONSE` by `request_id`, one `Thread` per inbound
  EXECUTE (§4.8 / §6.11), a `pending {request_id => Waiter}` map + `ConditionVariable` rendezvous,
  a per-connection write `Mutex`, `TCP_NODELAY` on every socket. The GVL is **released on blocking
  IO**, so the IO-bound peer is genuinely concurrent; the GVL does **not** make compound
  read-then-write atomic, so the §3.9 CAS store is an explicit `Mutex`-guarded critical section
  (proven by a 64-thread one-winner race). Ractors (true-parallel, share-nothing) are declined as
  the wrong tool at core; noted as the parallelism escape hatch (A-RUBY-004).

## Use

`require "entity_core"` brings in the whole `EntityCore` namespace. The stable contract is two tiers
(documented, not yet semver-frozen — Ruby has no module-private keyword, so the freeze is a
publish-prep documentation + test-relocation pass):

- **Tier 1 — model + codec + identity** (`EntityCore::Cbor`, `Entity`, `Envelope`, `PeerId`,
  `Identity`, `Signature`, `Hash`, the `Error` lattice): the value/codec island.
- **Tier 2 — full peer** (`EntityCore::Peer`, `Transport`, `Store`, `Capability`, `Handler`,
  `CoreTypes`): the §4.1 handshake, §6.5/§6.6 dispatch, the capability + store + type-registry
  surface, the TCP transport server/client.

Internal units (`Varint`, `Base58`, `Wire` framing, the `data/core_type_floor` render table) are
implementation detail and may churn without a semver bump. `exe/entity-core-peer` and
`exe/wire-conformance` are the S4 conformance/oracle **drivers** — test tooling, **not** part of the
published library surface (excluded from `spec.files`).

## Build, test, conformance (all in-container, sealed-offline)

The peer builds and conforms inside the keystone `ruby-toolchain` Podman image, fully offline
(`--network=none`) — the core peer has zero runtime gem deps, and Minitest + Rake ship with Ruby:

```sh
# unit + codec + agility + smoke regression (32 runs / 66 assertions)
podman run --memory=4g --memory-swap=4g --pids-limit=2048 --cpus=4 --rm --network=none -v "$PWD:/work:Z" \
  -w /work/protocol-generator/ruby entity-core-keystone/ruby-toolchain:latest \
  sh -c 'bundle install --local && bundle exec rake test'

# package the gem (produces entity_core_protocol-0.1.0.pre.gem)
gem build entity_core_protocol.gemspec

# full validate-peer --profile core gate (boots the Ruby peer + the Go oracle on
# the sealed loopback) — see run-s4.sh for the oracle-build isolation procedure
./run-s4.sh
```

`run-s4.sh` / `run-origination-core.sh` are the standing reproducers; they build the Go oracle from
the pinned `entity-core-go @75c532e` via `git archive` into a temp dir **outside** the oracle tree
(no writes ever land in `entity-core-go`) and run everything in one `--network=none` container.

## Versioning

The cohort release line is `0.1.0-pre`. RubyGems treats a literal `-` in a version string as a
`.pre.` separator (so `0.1.0-pre` canonicalizes to the malformed `0.1.0.pre.pre`); the
RubyGems-idiomatic pre-release spelling is the **dot-separated `0.1.0.pre`**, which `gem build`
emits and `Gem::Version#prerelease?` reports `true`. The gem coordinate is therefore `0.1.0.pre`
while this README/CHANGELOG carry the cohort `0.1.0-pre` label (A-RUBY-010 — the Ruby analogue of
Common Lisp's A-CL-010 ASDF wrinkle). Promotes to `0.1.0` when (a) S4 is fully green [met] **and**
(b) ≥1 external consumer confirms the peer [not yet met].
