# entity-core-protocol-kotlin

A native-Kotlin **core protocol peer** for the Entity Core protocol (V7), generated by the
[`entity-core-keystone`](../../README.md) `/entity-rosetta` pipeline. A **REACH peer** —
built for **Android / JVM-Kotlin ecosystem coverage** (V7 Layers 0–4 core), not as a
spec-discovery instrument: the JVM idiom was already exercised by the **Java peer**, so
Kotlin is corroboration-only on the spec axis (anything it surfaces anyway is high-signal,
and gets logged + escalated). It is, however, an **independent reader** of V7 — the codec is
**hand-rolled native Kotlin** (NOT interop into the Java peer's codec; see A-KT-001), and the
idiom axes are Kotlin-native (sealed-class `Result` error model + coroutines), so where a
value matches the Java peer it is by independent arrival from the shared JVM substrate, not by
inheritance.

## What it is

A native Kotlin/JVM peer with a near-zero-runtime-dependency core: canonical-CBOR (ECF) codec,
Ed25519 identity + deterministic signatures, capability authorization with chain attenuation +
§5.7 delegation caveats, the §4.1 handshake / §6.5–§6.6 dispatch machinery, the §9.5 53-type
registry, the §9.5a CORE-TREE get/put/CAS/delete surface, and the v7.73/v7.74 peer surface
(§6.13 register, §PR-8 granter frame, §6.9a owner-cap, §7a conformance handlers), plus the
v7.75 non-functional floor (§4.10 resource bounds, §7b concurrency). The **one** non-stdlib
runtime dependency is `kotlinx-coroutines-core` (the Kotlin-native concurrency primitive) — a
deliberate idiom-over-minimalism trade vs Java's literal zero deps (A-KT-003).

- **Codec:** hand-rolled canonical CBOR (`org.entitycore.protocol.codec`) — shortest-float
  minimization (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip), length-then-lex map-key sort
  on encoded key bytes, recursive major-type-6 tag rejection on decode, full 0..2⁶⁴−1 integer
  head-form. Kotlin's `UByte`/`ULong` (stable since 1.5) carry the uint64 head-form range
  **natively** — no `Long.compareUnsigned` / `ulong` / int63 dance the Java/C#/OCaml peers
  needed (a Kotlin-native ergonomic win on the number axis; A-KT-007). Base58 + the multicodec
  LEB128 varint are hand-rolled (neither is in the JDK).
- **Crypto:** Ed25519 sign/verify via the **JDK SunEC provider** (zero-dep, native);
  SHA-256 via `MessageDigest`. SunEC exposes no seed→public-key API, so the §1.5
  identity-multihash *raw public key* is derived by a pure-JDK RFC-8032 scalar-mult routine
  (`crypto/EdKeyDerivation.kt`), cross-checked against SunEC's own verifier and the
  cohort-canonical seed-`0x11` peer_id (A-KT-009, the same gap the Java peer hit). **Ed448 /
  SHA-384 agility** is a **deferred higher bar** for this reach peer (floor first): the JDK
  SunEC route (`NamedParameterSpec.ED448`) is documented + BouncyCastle (`bcprov-jdk18on` 1.80)
  is the opt-in cross-check, but the v0.1 target is the Ed25519 + SHA-256 **floor** (A-KT-002).
  The core build stays BouncyCastle-free.
- **Identity / peer_id:** §1.5 canonical-form construction (Ed25519 → `hash_type=0x00`
  identity-multihash, the digest IS the raw pubkey), following the §1.5 v7.65 table — already
  reconciled in the v7.75 body (the §7.4 contradiction OCaml/Zig/CL/Java surfaced is closed),
  pinned proactively (A-KT-004). The seed-`0x11` peer_id
  `2KHoAk7A5JmhygZJAdBua8iRD1CnBoJRfUBHgZeXNRTeFg` is **byte-identical to the Java / Common
  Lisp peers**. Lowercase `%02x` hex everywhere (dodges the A-CL-009 uppercase trap).
- **Error model (the headline divergence from Java):** Java chose **checked exceptions**;
  Kotlin has none. The recoverable seam is a **sealed-class** `EcfResult<T>` (Ok / Err) + a
  sealed `EntityError` hierarchy matched **exhaustively by `when`** (compiler-enforced — the
  static-rigor analogue of OCaml's `result` / Zig's error union, reached by Kotlin's own
  mechanism). Protocol status is carried as a value, never across an exception (the cohort's
  status-as-value invariant); a sealed `EntityError` variant maps to a status code
  (400 non_canonical_ecf / 401 / 403 / 413 payload_too_large / 400 chain_depth_exceeded) at the
  dispatcher boundary.
- **Concurrency (the JVM seam, Kotlin-native):** **kotlinx.coroutines** (structured
  concurrency) — one coroutine per connection on `Dispatchers.IO`, a reader coroutine demuxing
  `EXECUTE_RESPONSE` by `request_id` via `ConcurrentHashMap<requestId, CompletableDeferred<T>>`
  (the second deliberate divergence from Java's platform/virtual threads; A-KT-003). §7b
  store-safety via concurrent collections with atomic-per-key writes (A-KT-011). Public surface
  is BOTH a `runBlocking` blocking facade AND native `suspend` functions. All 5 §7b concurrency
  checks PASS.

## Build / test / run-conformance (in-container, offline)

All build/test/conformance runs happen in the pinned `kotlin-toolchain` Podman image
(`fedora:43` base + **Temurin JDK 21.0.10+7** SHA-256-pinned + **Kotlin 1.9.25** SHA-256-pinned
+ **Gradle 8.10.2** SHA-256-pinned). Gradle deps (kotlin-stdlib, kotlinx-coroutines,
kotlin-test, JUnit 5, opt-in BouncyCastle) are pre-fetched into the image's gradle caches at
container-build time, so every run is fully sealed-offline (`gradle --offline --no-daemon`,
`--network=none`). There is **no host Kotlin / Gradle dependency**.

```sh
# Unit tests — the S2 69/69 codec corpus + Ed25519 KATs + the 53-type byte-diff +
# the two-peer loopback smoke. Offline, --network=none.
podman run --memory=4g --memory-swap=4g --pids-limit=2048 --cpus=4 --rm --network=none -v "$PWD":/work:Z -w /work/protocol-generator/kotlin \
  entity-core-keystone/kotlin-toolchain:latest gradle --offline --no-daemon test

# S4 live-peer conformance (validate-peer --profile core). The Go oracle ELF and the
# JDK-21 host run together inside the kotlin-toolchain image, sharing one loopback, offline.
./protocol-generator/kotlin/run-s4.sh

# origination-core (§6.11 reentry over real two-peer TCP, Go entity-peer reference): 3/3
./protocol-generator/kotlin/run-origination-core.sh
```

The S4 oracle (`output/s4-oracles/validate-peer`) is a `CGO_ENABLED=0` Go ELF built from a
disposable `git archive` of the READ-ONLY sibling `entity-core-go` at the pinned ref, via
`tools/oracle-bootstrap.sh` (`tools/oracle-pin.env` records the pin `e8524ed` +
`core_gate_sha256`). A rebuilt oracle reproducing that hash proves its core surface is
identical to what the cohort converged against. See
[`status/CONFORMANCE-REPORT.md`](status/CONFORMANCE-REPORT.md) §Provenance.

## Use

Public surface is the `org.entitycore.protocol` package tree. Two tiers:

```kotlin
import org.entitycore.protocol.codec.*   // Tier 1 — codec island
import org.entitycore.protocol.crypto.*  // Tier 1 — identity + signatures
import org.entitycore.protocol.peer.*    // Tier 2 — full peer + transport

// Tier 1 — encode/verify ECF (no peer)
val bytes: ByteArray = EcfCodec.encode(entity.toCbor())   // canonical ECF bytes
val id = Identity.ofSeed(seed)                             // Ed25519 identity + §1.5 peer_id

// Tier 2 — full peer (suspend surface; a runBlocking facade is also provided)
val peer = Peer.create(seed, openGrants = false)
Transport(peer).acceptLoop(7777)                          // coroutine per connection
```

Internal units (varint, base58, wire framing, the type-registry render table) are
implementation detail and may churn without a semver bump. An explicit public-surface freeze
(an `@PublishedApi` / module-visibility lock) is a mechanical publish-prep pass, deferred until
the surface is frozen against a first external consumer — the honest `0.1.0-pre` state (mirrors
the Java / OCaml / Zig deferral).

## Distribution (when published)

Kotlin's registry is **Maven Central** (Sonatype Central Portal) — Kotlin libraries publish
like any JVM artifact, via the vanilla Gradle `maven-publish` + `signing` plugins (no
third-party publish plugin, to keep the supply chain minimal). The
`build.gradle.kts` `publishing {}` / `signing {}` blocks are authored with the
`org.entitycore:entity-core-protocol-kotlin:0.1.0-pre` coordinates, POM metadata
(Apache-2.0 license, `Entity Core Protocol contributors` developer), and the publish
credentials / repository URL left as **explicit publish-time `TODO`s**. "Publishing" is
`gradle publish` against a **verified `org.entitycore` reverse-DNS namespace** — the namespace
verification is a one-time operator step that gates the first deploy (A-KT-005), so Maven
Central publishing is **deferred**, not done here. See
[`status/PHASE-S5.md`](status/PHASE-S5.md) §Operator-handoff.
