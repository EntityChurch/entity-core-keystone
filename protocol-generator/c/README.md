## What it is

A native C peer whose **only third-party runtime dependency is libsodium** (Ed25519 + SHA-256):
the canonical-CBOR (ECF) codec, Base58, the multicodec LEB128 varint, content-hash, peer-id, the
§4.1 handshake / §6.5–§6.6 dispatch machinery, capability authorization with chain attenuation +
§5.7 delegation caveats, the §9.5 53-type registry, the §9.5a CORE-TREE get/put/CAS/delete surface,
the §10.1 core-register gate, the §4.10 resource-bounds checks, and the §7b concurrency gate are all
hand-rolled in `src/`. No CBOR library (none gives ECF canonicality), no FFI to the sibling C codec
(that codec is kept only as a free byte-for-byte cross-check oracle — A-C-005).

- **Codec:** hand-rolled canonical CBOR (`src/ecf.c`, encode + decode in one translation unit) —
  shortest-float minimization (f16 ⊂ f32 ⊂ f64, narrowest bit-exact round-trip), length-then-lex
  map-key sort on encoded key bytes (`qsort` + a length-then-`memcmp` comparator), recursive
  major-type-6 tag rejection at any depth, definite-length only. **Full uint64 / −2⁶⁴ head-form via
  native `uint64_t`** — the cleanest int story in the cohort alongside Zig (no `ulong`/int63
  special-casing). Base58 (`src/base58.c`) + LEB128 varint (`src/varint.c`) hand-rolled.
- **Crypto:** Ed25519 + SHA-256 via **libsodium 1.0.22** (`crypto_sign_*` / `crypto_hash_sha256`,
  `src/crypto.c`); the all-zero-seed → RFC-8032 TEST-1 pubkey KAT passes. **Ed448 / SHA-384 agility
  is deferred** (A-C-001) — libsodium has no Ed448; the §9.1 floor (Ed25519 + SHA-256) is fully
  native and is the only path the corpus exercises. Nonces come from the libsodium CSPRNG
  (`ec_random_bytes`, A-C-010 — the clock-nonce replay fix).
- **Identity / peer_id:** §1.5 canonical-form construction (Ed25519 → `hash_type=0x00`
  identity-multihash, digest *is* the raw 32-byte pubkey), per the §1.5 v7.65 table, **not** the
  stale §7.4 SHA-256 pseudocode (A-C-009/peer-id, the 5th+ spec-first corroboration). Lowercase
  `%02x` hex everywhere (naturally avoids the A-CL-009 uppercase address-space-path trap; pinned in
  `profile.toml [idiom].hex_lowercase`). The seed-`0x11` peer_id
  `2KHoAk7A5JmhygZJAdBua8iRD1CnBoJRfUBHgZeXNRTeFg` is **byte-identical to the cohort**.
- **Error model:** the **return-code + out-param** idiom — every fallible function returns an
  `ec_status` (`EC_OK == 0`; negative = a specific failure enum) and writes its result through an
  out-pointer. Protocol status is carried as a value, never smuggled across an error return.
  §5.2 request verification is a three-way verdict (ALLOW / AUTHN_FAIL→401 / AUTHZ_DENY→403) + the
  §5.5 unresolvable-grantee→401 carve-out.
- **Memory:** explicit `malloc`/`free`, `goto`-cleanup error paths, documented caller-frees
  ownership (every allocating API names its matching `ec_*_free()`). Shared entity refcounts are
  **`atomic_int`** (A-C-009 ⚑ — the headline arch-bound finding: a plain-`int` refcount raced under
  the live §7b concurrency load and freed a still-referenced entity; only a no-GC manual-memory peer
  surfaces this). The decode tree owns its node copies (the input buffer need not outlive it).
- **Concurrency (the C seam):** POSIX **pthreads** — one reader thread per connection demuxing
  `EXECUTE_RESPONSE` by `request_id`, one thread per inbound EXECUTE (§4.8 / §6.11), a
  `pthread_rwlock_t` guarding the content store, a per-connection write mutex, `TCP_NODELAY` on
  every connection socket. All 5 §7b concurrency checks PASS.

## Consume

The stable public ABI is the single umbrella header **`include/entity_core/protocol.h`** plus the
`ec_*` symbols (`-fvisibility=hidden` + an `EC_API` export macro hide everything else; libsodium's
symbols are localized so an embedder's own libsodium does not collide). Two link forms ship:

```sh
# Static archive — fully self-contained (libsodium statically linked in): the primary C artifact.
cc myapp.c -I/usr/local/include /usr/local/lib/libentity_core_protocol.a -lpthread -o myapp

# Shared object — depends on the system libsodium.so (A-C-006; see Packaging note below).
cc myapp.c $(pkg-config --cflags --libs entity-core-protocol) -o myapp
```

```c
#include <entity_core/protocol.h>

/* Tier 1 — the codec/crypto island (the public ABI). */
ec_crypto_init();
ec_value *e   = ec_map();                          /* build an ECF value tree */
uint8_t  *buf; size_t n;
ec_ecf_encode(e, &buf, &n);                        /* canonical ECF bytes */
ec_value *back;
ec_ecf_decode(buf, n, &back);                      /* structural decode (caller frees) */
uint8_t pid_pub[32]; char *pid;
ec_peer_id_from_pubkey(0x01, pub, 32, &pid);       /* §1.5 canonical peer_id */
ec_value_free(e); ec_value_free(back); free(buf); free(pid);
```

The **full peer + transport** (Tier 2 — `Peer`, `Transport`, the store + capability surface) lives
in `src/*.h` (`peer_internal.h`, `transport.h`, `capability.h`) and is driven via the **`entity-peer-c`
host binary** (`make` → `entity-peer-c --port N [--debug-open-grants] [--validate]`, emits
`LISTENING …`). That peer-layer surface is **not** part of the frozen public header at `0.1.0-pre` —
exposing it as a stable Tier-2 ABI is a mechanical publish-prep pass, deferred until the surface is
frozen against a first external consumer (the honest `-pre` state; mirrors the OCaml `.mli` / Zig
`root.zig` deferral). Internal units (varint, base58, wire framing, the type-registry render table)
may churn without a semver bump.

## Distribution (when published)

C has **no central package registry** (no crates.io/npm/Maven). "Publishing" is a versioned
**source tarball** (the autotools/Make convention) carrying the static + shared lib and a
`pkg-config` (`.pc`) file, consumed downstream via distro packaging or vendoring — the most
decentralized stance in the cohort, sibling to Zig's. `make dist` produces
`entity-core-protocol-c-0.1.0-pre.tar.gz` (sources + `entity-core-protocol.pc.in` + LICENSE +
headers); `make install` lays down the `.a`/`.so` + header + the rendered `.pc` under `$(PREFIX)`.
There is no publish command and nothing to submit to. See
[`status/PHASE-S5.md`](status/PHASE-S5.md) §Operator-handoff and
[`status/ARCHITECTURE-REVIEW.md`](status/ARCHITECTURE-REVIEW.md) §Publishing-options for the
in-repo-vs-standalone-repo decision surface (operator-decides; not acted on by the pipeline).

**Packaging note (A-C-006):** Fedora's `libsodium.a` is built **without `-fPIC`**, so it cannot be
linked into the peer `.so` (an `R_X86_64_PC32` relocation error). The `.a` consumer path statically
links the distro `libsodium.a` (fully self-contained — the profile's preferred shape); the `.so`
links the shared `-lsodium` and so carries a normal `libsodium.so` `NEEDED`. A self-contained `.so`
would need a `-fPIC` libsodium built from source to bundle privately — an S5/release concern, not a
conformance one. The static-`.a` path already meets the self-contained goal.
