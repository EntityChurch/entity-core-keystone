## What it is

A pure-Haskell peer: a canonical-CBOR (ECF) codec, Ed25519/Ed448 identity + signatures, capability
authorization with chain attenuation, and the §4.1 handshake / §6.5 dispatch machinery. **One
pinned Hackage dependency on the wire path — `crypton`** (for crypto); CBOR / Base58 / varint are
hand-rolled in-repo; transport/store use the GHC-boot `network` / `stm` / `containers` set. The
dependency closure is pinned by a committed `cabal.project.freeze` derived from Stackage LTS 23.27.

- **Codec — hand-rolled canonical ECF, zero CBOR dependency** (`EntityCore.Codec.CBOR`). The
  shortest-float ladder (f16⊂f32⊂f64, ENTITY-CBOR-ENCODING §2.2 Rule 4), length-then-lexicographic
  map-key sort over the *encoded* key bytes (Rule 2), recursive major-type-6 tag rejection on
  decode (§6.3), full `Word64`/n-int range, raw-byte fidelity. No platform CBOR library gives
  ECF's exact guarantees (the 8th confirmation of that cross-peer finding) — so native is the
  lighter path.

- **Crypto — fully native, no FFI, no defer (the headline data point).** `crypton` supplies
  Ed25519 **and Ed448** *and* SHA-256/384/512 from the same audited C-backed library. **Haskell is
  the first peer with native full crypto-agility including Ed448** — the §1.5 `key_type 0x02`
  higher bar (57-byte pubkey, 114-byte RFC-8032 signature) is reached **in the default build**, no
  opt-in agility sub-library and no system-package dependency. (OCaml hit the native Ed448 gap →
  hybrid FFI; Zig/Swift deferred Ed448 to an FFI path; Haskell needs neither.) Honest scope: the
  oracle's `crypto_agility` category is satisfiable by the FFI-deferred peers too — it exercises
  peer-id/key-type handling at the protocol surface, not a live core-gate Ed448 signature; the
  native-vs-FFI distinction lives at the test-corpus / library-availability layer.

- **Pure codec + monadic-IO boundary.** The codec is a *total pure function* returning
  `Either CodecError a` — there is no exception path in the codec at all; a decode/encode failure
  is a `CodecError` *value*. `Control.Exception` / `throwIO` / `bracket` appear **only** at the
  transport edge (sockets). The capability verdict is likewise a pure function over a one-time store
  snapshot (no IO in the verdict path). (A-HS-001.)

- **Laziness / strictness discipline (A-HS-002).** Haskell is lazy by default — the one axis where
  an *implementation* finding could surface in a byte-exact codec. The probe result: **purity makes
  the codec laziness-immune** — a fully-lazy encoder produces byte-identical output (encoding is
  pure), so the strictness work (strict `ByteString` accumulator forced from a `Builder`,
  bang-materialized map-sort keys, `StrictData` fields, `deepseq` at the API edge) is a *space-leak
  / determinism* discipline, **not a correctness one**. The single genuine correctness trap is a
  *string-length* one, loudly enforced: a text-string's CBOR length is the **UTF-8 byte count**
  (`Data.Text.Encoding.encodeUtf8`), **never** `Data.Text.length` (code points) — `"héllo☃"` is 6
  code points but 9 bytes. Reusable guidance for any future lazy/non-strict language.

- **Concurrency — STM + GHC green threads (the §7b story).** The live store is an STM `TVar`
  mutated inside `atomically`; one `forkIO` green thread per connection; request_id↔reply
  correlation via an STM `TVar` demux map that `retry`s on an empty slot. This is a **3rd
  data-race-free store shape** after the Elixir actor (message-serialized) and Swift actor
  (await-serialized) — here *transactional*: the exact store race that crashed Zig (HashMap
  double-free) and Common Lisp (raced `gethash` → 500s) **cannot occur** by construction. And the
  GHC `-threaded` RTS **sidesteps Swift's blocking-syscall trap**: the IO manager multiplexes a
  blocking `recv` over `epoll`, so a parked read *yields its capability* rather than starving the
  scheduler. Both are why the §7b concurrency gate passed 5/5 with **no per-check fix** (the legs
  that RED'd the rest of the cohort — sustained C=16×K=10000 load, concurrent reentry — pass
  structurally). `TCP_NODELAY` on every socket.

- **Integers.** The ECF head-form carrier holds the full 0..2⁶⁴−1 range as `Word64` (no native-int
  trap — `Word64` is exactly 64-bit unsigned).

## Use

The public module surface is re-exported under `EntityCore.*`.

```haskell
import qualified Data.ByteString as BS

-- Codec island (the shared-data-library tier): encode/verify ECF.
import EntityCore.Codec       (encode, decode)          -- Value <-> ByteString, pure
import EntityCore.Model       (makeEntity)              -- {type,data,content_hash}
import EntityCore.ContentHash (contentHash)             -- varint(fmt) ++ HASH(ECF{...})
import EntityCore.Identity    (identityOfSeed, signEntity, idPeerId)

-- An identity (Ed25519) + its §1.5 peer_id, derived from a 32-byte seed:
demo :: BS.ByteString -> Either String ()
demo seed = do
  ident <- identityOfSeed seed       -- Right Identity ; idPeerId ident :: Text (Base58)
  let e = makeEntity "system/peer" (error "ECF Value for the data field")
  pure ()
```

```haskell
-- Full peer (the higher-bar tier): boot a peer, serve a port.
import EntityCore.Peer       (createPeer)
import EntityCore.SeedPolicy (standardPolicy)            -- conformant §4.4-floor default
import EntityCore.Transport  (listenOn, acceptLoop)

bootPeer :: BS.ByteString -> IO ()
bootPeer seed = do
  -- createPeer seed seedPolicy conformanceHandlers
  --   seedPolicy  : standardPolicy (conformant) | SeedPolicyDebugOpen (degenerate default->*)
  --   conformance : True enables the §7a system/validate/* handlers (off by default)
  peer        <- createPeer seed standardPolicy False
  (sock, port) <- listenOn 7777
  acceptLoop peer sock
```

Or run the bundled standalone host (the conformance target):

```
host [--port N] [--validate] [--owner-identity ...] [--seed-policy FILE] [--debug-open-grants]
```

`--validate` enables the §7a `system/validate/{echo,dispatch-outbound}` conformance handlers (off
by default — `dispatch-outbound` is a standing originator, never live in production).
`--debug-open-grants` is **deprecated** (v7.74 §6.9a; selects the degenerate `default → *` seed
policy).

Every build/test/conformance run happens in the pinned `ghc-toolchain` Podman image (fedora:43
base; GHC 9.8.4 / cabal-install 3.14.2.0), sealed-offline against a warm cabal store + the
committed freeze (A-HS-005). Warnings-as-errors (`-Werror`) is behind the manual `dev` flag
(off in the distributed build, on for the hygiene loop).

  -e CABAL_DIR=/work/protocol-generator/haskell/.cabal-home \
  -v "$PWD":/work:Z -w /work/protocol-generator/haskell \
  entity-core-keystone/ghc-toolchain:latest \
